# Web Media Parser

**Desktop-приложение для интеллектуального парсинга и загрузки медиафайлов с веб-сайтов.**

Автоматически обнаруживает изображения и видео, отличает полноразмерные версии от миниатюр, обходит защиты (cookie-consent, age-gate, JS-редиректы) и адаптируется к структуре конкретных сайтов через систему паттернов.

---

## Возможности

### Умное обнаружение медиа
- **Full-size из thumbnails** — преобразует URL миниатюр в полноразмерные изображения через встроенные паттерны и правила Imagus (включая JS-правила)
- **Приоритизация URL** — медиа-ссылки обрабатываются первыми, навигационные — в последнюю очередь
- **Фильтрация «мусора»** — автоматический пропуск иконок, баннеров, tracking-пикселей, SVG и GIF-анимаций
- **Распознавание lazy-load** — извлекает изображения из `data-src`, `srcset`, inline-CSS и JavaScript-кода
- **Ограничение на домен** — макс. 2 одновременных запроса к одному домену (парсинг + скачивание)

### Обход защит и динамического контента
- **Cookie-consent** — автоматическая установка согласий на использование cookies
- **Age-gate / Gateway** — нажатие кнопок «Мне есть 18», «I agree» и аналогичных
- **JS-редиректы** — обнаружение и переход по JavaScript-редиректам (до 5 редиректов)
- **HTTP 429** — автоматический backoff с учётом `Retry-After` заголовка
- **Fallback aiohttp → requests** — если сервер блокирует асинхронные запросы, используется sync `requests` с полным набором браузерных заголовков

### Система паттернов
- **Встроенные паттерны** — готовые правила для популярных сайтов
- **Custom patterns (JSON)** — собственные правила с `image_transformations`
- **Imagus Sieve** — загрузка правил Imagus; JS-правила конвертируются в Python callables
- **Page only режим** — скачивание только с указанной страницы без обхода сайта

### Очередь задач
- **Мультизадачность** — очередь с паузой/продолжением каждой задачи
- **Page only режим** — чекбокс для скачивания только с текущей страницы
- **Автозапуск** — следующая задача запускается автоматически при завершении текущей
- **Экспорт/Импорт CSV** — выгрузка и загрузка очереди из CSV-файла
- **Очистка истории** — удаление очереди и session-файлов (скачанные файлы сохраняются)

### Загрузка
- **Многопоточность** — настраиваемое число потоков парсера и загрузчика
- **Multi-threaded download** — загрузка одного файла несколькими потоками (с `Accept-Ranges`)
- **Общая HTTP-сессия** — переиспользование TCP/TLS-соединений
- **Контроль скорости** — ограничение пропускной способности
- **Дедупликация** — пропуск уже загруженных URL
- **Атомарное сохранение** — state.pkl через temp + fsync + rename (защита от аварий)

### Портабельность
- **onedir сборка** — все файлы приложения в одной директории
- **Относительные пути** — ресурсы, настройки, session — всё рядом с exe
- **Скачивание в выбранную папку** — путь запоминается между запусками

### Поддержка форматов
- **Изображения:** JPEG, PNG, WebP, AVIF, BMP, TIFF
- **Видео:** MP4, WebM, OGG, MOV, MKV, AVI, FLV, M4V
- **Аудио:** MP3, WAV, AAC, FLAC, OPUS
- **Streaming:** M3U8 (HLS), MPD (DASH)
- **API:** JSON-эндпоинты с извлечением медиа из вложенных структур

### Интерфейс
- **PySide6 GUI** — тёмная тема, лог с фильтрацией по уровням
- **Пауза / Продолжить / Стоп** — полный контроль над процессом
- **Прогресс-бары** — общий прогресс и прогресс текущего файла
- **Вывод лога в файл** — опционально через настройки (Settings → Logging)
- **Защита настроек** — валидация всех полей при загрузке (anti-tamper)

## Установка

### Системные требования
- **Python** 3.8+
- **ОС:** Windows 10/11 (Linux/macOS через кроссплатформенные зависимости)

### Из исходников

```bash
git clone https://github.com/Sucotasch/web-media-parser.git
cd web-media-parser
pip install -r requirements.txt
python main.py
```

### Сборка портабельного пакета

```bash
python build_exe.py
```

Результат: `dist/WebMediaParser/` — директория с `WebMediaParser.exe` и всеми зависимостями.

Структура:
```
dist/WebMediaParser/
├── WebMediaParser.exe
├── Imagus_sieve_2026.04.01_849.json
├── sessions/          (создаётся при первом запуске)
└── _internal/
    └── resources/
        ├── dark_theme.qss
        ├── domain_blocklist.txt
        └── patterns/site_patterns.json
```

## Использование

1. **Запустите** приложение (`python main.py` или `WebMediaParser.exe`)
2. **Вставьте URL** сайта, с которого нужно собрать медиа
3. **Выберите папку** для сохранения файлов (путь запоминается)
4. **Настройте параметры** через *Settings* (по умолчанию оптимальны)
5. Нажмите **Start**

### Дополнительно
- **☑ Page only** — скачивает только медиа с указанной страницы, не обходит сайт
- **Export CSV** — выгрузка очереди задач в CSV
- **Import CSV** — загрузка задач из CSV (восстанавливает завершённые, скачивает новые)
- **Clear History** — очистка очереди и session-файлов

### Структура загрузки

```
<download_dir>/
└── example_com_20260412_1630/
    ├── images/
    │   └── photo_001.jpg
    ├── videos/
    │   └── clip_001.mp4
    └── ...
```

## Настройки

### Parsing
| Параметр | По умолчанию | Описание |
|---|---|---|
| Search Depth | 3 | Глубина перехода по ссылкам (0 = только стартовая) |
| Page Limit | 1000 | Макс. кол-во файлов для скачивания (0 = без лимита) |
| Page Timeout | 30 сек | Таймаут загрузки страницы (5–300) |
| Page Limit | 1000 | Остановка после скачивания N файлов |
| Stay in Domain | ✓ | Не уходить на другие домены |
| Process JavaScript | ✓ | Анализировать JS-контент и lazy-load |
| Use Image Patterns | ✓ | Преобразовывать thumbnail → fullsize |
| Bypass Cookie Consent | ✓ | Обходить баннеры cookies |
| Bypass JS Redirects | ✓ | Переходить по JS-редиректам |
| Filter Hidden Links | ✓ | Игнорировать скрытые ссылки (бот-ловушки) |

### Filters
| Параметр | По умолчанию | Описание |
|---|---|---|
| Min. Image Width | 100 px | Минимальная ширина (из HTML-атрибутов) |
| Min. Image Height | 100 px | Минимальная высота (из HTML-атрибутов) |
| Min. Image Size | 40 KB | Минимальный размер файла (проверяется через HEAD) |
| Min. Video Size | 1000 KB | Минимальный размер видео |
| Stop Words | список | URL, содержащие эти слова, пропускаются |

### Performance
| Параметр | По умолчанию | Описание |
|---|---|---|
| Parser Threads | 4 | Потоки парсинга страниц |
| Downloader Threads | 8 | Потоки загрузки файлов |
| Threads per File | 1 | Потоков на загрузку одного файла (до 8) |
| Max Speed | 0 (без лимита) | Ограничение скорости (KB/s) |

### HTTP
| Параметр | По умолчанию | Описание |
|---|---|---|
| User-Agent | Chrome 130 | Строка User-Agent |
| Referer | auto | Политика Referer (`auto` / `origin` / `none`) |
| Accept-Language | en-US | Язык ответов |
| Timeout | 30 сек | Таймаут сетевого запроса |
| Retry Count | 3 | Повторные попытки при ошибке |
| Proxy | — | Прокси в формате `host:port` |

### Logging
| Параметр | По умолчанию | Описание |
|---|---|---|
| Enable logging to file | Выкл | Запись лога в текстовый файл |

### Ключевые механизмы

**Приоритизация URL** — многофакторный скоринг:
- Прямые медиа-URL ×25, image-linked ×20
- Совпадение пути с начальной страницей ×3+ за компонент
- Домашние страницы и навигация ×0.005
- Репутация домена (сколько медиа найдено)

**Domain concurrency limit** — макс. 2 одновременных запроса к одному домену (парсинг + скачивание).

**Карантин** — домены с 3+ ошибками временно блокируются. Повторная попытка 1 раз, затем URL отбрасывается. Probation timeout 5 сек.

**Natural completion** — автозавершение при пустых очередях (>5 сек без активности).

**Page limit** — остановка после скачивания N файлов (считает реальные загрузки, не посещённые страницы).

## Устранение проблем

| Проблема | Решение |
|----------|---------|
| Медиа не обнаруживается | Включите **Process JavaScript**, добавьте Imagus Sieve |
| Низкая скорость | Увеличьте **Downloader Threads** до 16–32 |
| Сайт блокирует запросы | Настройте **Proxy**, увеличьте **Timeout** |
| 429 Rate Limit | Приложение автоматически делает backoff с Retry-After |
| Приложение «зависло» | Проверьте лог, используйте **Stop** (немедленная остановка) |
| Дубликаты при Resume | Автоматически фильтруются (проверка downloaded_files) |
